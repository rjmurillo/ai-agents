Verified against production examples: Claude Code v2.1+ auto-discovers `hooks/hooks.json` from any installed plugin without requiring an explicit `hooks` key in `plugin.json`. Reference: `~/.claude/plugins/cache/context-mode/.claude-plugin/plugin.json` has no `hooks` key but ships `hooks/hooks.json` with active event handlers. Same pattern in `claude-plugins-official/security-guidance`.

Adding `"hooks": "./hooks/hooks.json"` would be redundant. The wrapping bug that would have prevented hooks from loading was the missing `"hooks"` key *inside* `hooks.json`, fixed in 78bd2444 + verified in 58a03bcd. Plugin consumers will receive all 7 hook events via auto-discovery.
