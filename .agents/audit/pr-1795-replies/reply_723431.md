By design. The session log is committed mid-session (per AGENTS.md commit cadence) and the `sessionEnd` MUST items are filled in at the actual session end via the `session-end` skill, which auto-populates evidence (commit SHA, lint results, validation status) and re-validates. Until then they correctly read `Complete: false`.

The `Session Protocol Validation` workflow only blocks merge if MUST items are still incomplete *at merge time*, not while the session is in flight. Will run `session-end` before requesting merge.
