Addressed in db0971b3. `_validate_hooks()` now accepts a `manifest_dir` parameter. When `hooks` is a string ref to `*.json` and the referenced file resolves under the plugin root, it is loaded and its contents are validated:

- Must be a JSON object
- May be wrapped under `"hooks"` key (canonical hooks.json shape) or be the bare events object
- Each event name must be in `VALID_HOOK_EVENTS`
- Each entry must conform to matcher-group structure (`type=command`, `command` string)

If the referenced file does not exist, the path-shape check still applies but the content check is skipped (so dynamic/runtime-resolved paths don't fail the gate).
