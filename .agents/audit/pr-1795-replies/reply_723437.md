The RCA says the prior failure was caused by `agents`/`skills`/`commands` being declared as **arrays of directory paths** with the broken shape `["./agents"]` plus `hooks` as a dict-of-directories. The validator accepts string-or-array-of-strings (the documented Anthropic schema), with the new `_validate_relative_path()` enforcing the `./` prefix and `..` ban — which is exactly what catches the dict shape and the missing-prefix shape.

What actually broke install in PR #1773 was *not* `agents: ["./agents"]` per se (that survives the schema check); it was the combination of (a) `hooks` being a dict-of-directories which Claude Code rejects with "Invalid input", and (b) the agent files at the plugin root not being under any `./agents/` directory — so removing the key broke discovery. Both are now covered: hooks dict-of-strings rejected (`test_regression_hooks_as_dict_of_strings_rejected`), agents path repaired with `"./"` (`ba7778af`).
