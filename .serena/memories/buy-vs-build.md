# Buy vs Build Decision Framework

**Created**: 2026-01-10
**Category**: Strategic Decision

## Overview

Framework for evaluating vendor solutions versus in-house development.

## When to Build

| Factor | Build If... |
|--------|-------------|
| Core business | Software is core to business needs |
| Differentiation | Competitive advantage required |
| Control | Need 100% control over functionality |
| Resources | Team has capacity without additional investment |

## When to Buy

| Factor | Buy If... |
|--------|----------|
| Time to value | Implementation in weeks vs months matters |
| Team focus | Team should work on higher-impact projects |
| Cost efficiency | Vendor economies of scale reduce total cost |
| Standard capabilities | Not differentiating features |

## Risks

### Build Risks

- No assurance product meets all requirements
- Full cost burden: buildout, support, upgrades, migrations
- Can add up to hundreds of thousands of dollars

### Buy Risks

- Vendor dependence
- Platform lock-in
- Limited customization

## Hybrid Approach

Often the best solution: Build differentiating components, buy commodity capabilities.

## Decision Metrics

Organizations using structured decision matrices achieve 35% higher project success rates.

Key factors:

- Available budget
- Business goals
- Future scalability
- Expected product lifetime
- Available market solutions
